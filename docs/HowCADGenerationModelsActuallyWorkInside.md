# How CAD generation models actually work inside

**Neural CAD generation has split into two fundamentally different pipelines—direct neural prediction of geometry and topology versus LLM-based code generation—each with distinct strengths, failure modes, and architectural machinery.** The direct approach trains specialized networks (transformers, diffusion models, VQ-VAEs) to output tokenized CAD representations that must be reconstructed into valid solids, achieving 46–83% validity rates depending on method. The code generation approach leverages pretrained LLMs to produce CadQuery or OpenSCAD scripts, offloading topological consistency to battle-tested CAD kernels and achieving 90–100% compilation rates. This report traces both pipelines end-to-end: from input encoding through tokenization, generation architecture, and output reconstruction, with exact dimensions, loss functions, and training parameters drawn from the primary papers.

## How text and images become conditioning signals

CAD generation models convert natural language and visual inputs into dense vectors that steer the generation process through three distinct injection strategies: cross-attention, adaptive normalization, and token concatenation.

**Text2CAD** (Khan et al., NeurIPS 2024 Spotlight) uses a frozen BERT encoder with **d_p = 1024** output dimension and maximum **512 word tokens**. The raw BERT embeddings pass through a trainable adaptive layer that projects them into a CAD-compatible representation (also 1024-dim), bridging the semantic gap between language and geometry. The CAD decoder is an 8-block transformer with **256-dim** hidden states and **8 attention heads**. Critically, the first two decoder blocks use only self-attention—cross-attention between CAD queries and text keys/values activates starting at block 3. This staged injection lets early layers establish basic structural features before text conditioning refines them. The cross-attention follows the standard formulation: Q is projected from CAD hidden states via W_Q ∈ ℝ^(256×d_q), while K and V are projected from the adapted text embedding via W_K and W_V. Training uses cross-entropy loss with teacher forcing over **272 maximum CAD tokens**, running for **160 epochs** on a single A100-80GB in roughly two days. The dataset comprises ~170K DeepCAD models with ~660K multi-level text annotations generated through a two-stage pipeline: LLaVA-NeXT produces abstract shape descriptions from multi-view renders, then Mixtral-50B generates four annotation levels (abstract through expert).

**CAD-MLLM** (Xu et al., 2024) takes a different approach entirely, using **Vicuna-7B** fine-tuned with LoRA as its backbone. Multi-view images (8 fixed perspectives) are encoded by a **frozen DINOv2** vision encoder, compressed through a perceiver-based cross-attention module with learnable query tokens, then projected into the LLM's embedding space. Point clouds pass through a **frozen Michelangelo** encoder with linear projection. All modality tokens—text, visual, and point embeddings—are simply concatenated as prefix tokens in the LLM input sequence, relying on self-attention to integrate cross-modal information. Training follows a curriculum: text descriptions first, then point clouds, then multi-view images, with random modality combinations for diversity. The Omni-CAD dataset provides **~450K instances** with all three modalities.

A third strategy appears in **CadVLM** (ECCV 2024), which uses a ViT-MAE-base vision encoder paired with CodeT5+'s 770M-parameter text encoder-decoder. Here, alignment happens through an **image-text contrastive loss (ITC)** that pulls visual and textual feature spaces together, combined with image decoding loss and language modeling loss trained jointly. This contrastive alignment approach achieved **23.8% sketch accuracy** versus Vitruvion's 3.0%, demonstrating the value of explicit cross-modal alignment over simple concatenation for sketch-level tasks.

## Turning continuous geometry into discrete tokens

The central challenge of neural CAD generation is converting inherently continuous geometry—coordinates, angles, surface parameters—into formats that neural networks can consume and produce. Three representation strategies dominate the field, each encoding CAD data at a different level of abstraction.

**DeepCAD's sketch-and-extrude sequences** (Wu et al., ICCV 2021) represent CAD models as flat command sequences with 6 command types: ⟨SOL⟩ (start-of-loop), line, arc, circle, extrude, and ⟨EOS⟩ (end-of-sequence). Each command carries a **unified 16-parameter vector** covering sketch parameters (endpoint x,y; arc sweep angle α; CCW flag f; circle radius r), extrusion parameters (sketch plane orientation θ,φ,γ; origin p_x,p_y,p_z; scale s; extrusion distances e_1,e_2), boolean type b ∈ {new, join, cut, intersect}, and extrude type u ∈ {one-sided, symmetric, two-sided}. Unused parameters for a given command type are set to −1. All continuous parameters undergo **8-bit quantization to 256 levels** after normalization (the entire model is scaled within a 2×2×2 cube; each sketch profile is normalized into a unit square). Sequences are padded to a fixed length of **N_c = 60 commands**. Each command is embedded as the sum of three components: a command type embedding (W_cmd ∈ ℝ^(256×6)), a parameter embedding (each of 16 parameters encoded as 257-dim one-hot—256 levels plus "unused"—then combined via two linear layers), and a learned positional embedding.

**SkexGen's disentangled codebooks** (Xu et al., ICML 2022) factor the representation into three independent aspects. The topology codebook (500 entries, 4 code-tokens, 256-dim embeddings) captures loop and curve structure. The geometry codebook (1,000 entries, 2 code-tokens) stores curve coordinates. The extrusion codebook (1,000 entries, 4 code-tokens) encodes 3D parameters. SkexGen uses **6-bit quantization** (64 levels), which the authors found sufficient for most CAD models. Each codebook is maintained via VQ-VAE with **EMA updates at decay rate γ = 0.99**: cluster counts and embedding sums are exponentially averaged, then codebook vectors are set to the ratio m_i/N_i. The commitment loss weight is **β = 0.25**, and code quantization is skipped during the initial 25 epochs for training stability. The total representation of a CAD model is just **10 discrete codes**—dramatically more compressed than DeepCAD's 60×17 token array. A separate autoregressive code selector generates these 10 codes at inference using nucleus sampling with **top-p = 0.95**.

**BrepGen's hierarchical B-rep latents** (Xu et al., SIGGRAPH 2024) operate at an entirely different level: directly on boundary representations. Following UV-Net's approach, B-spline surfaces are uniformly sampled on a **32×32 UV-parameter grid** to produce face shape features F_s ∈ ℝ^(32×32×3), while edges are sampled at 32 points to produce E_s ∈ ℝ^(32×3). A 2D convolutional UNet VAE compresses face grids by a factor of 8 into **F_z ∈ ℝ^(4×4×3) = 48 dimensions**. A 1D convolutional UNet compresses edges into **E_z ∈ ℝ^(4×3) = 12 dimensions**, concatenated with 6D vertex coordinates to form 18D joint edge-vertex latents. Both VAEs use MSE reconstruction loss with KL regularization weighted at **λ_KL = 0.02**.

**Why quantization-as-classification works.** DeepCAD's ablation provides the clearest evidence: replacing 256-level classification with continuous regression (MSE loss) degraded median Chamfer Distance from **0.787×10⁻³ to 2.142×10⁻³**—a 2.7× deterioration. The paper explains that classification naturally captures multi-modal distributions over parameter values, allowing the model to snap to discrete geometric constraints (exactly 90° angles, exactly parallel edges) that regression with a unimodal Gaussian mode systematically blurs. However, Img2CAD (2024) found the opposite effect in a conditional factorization setting where structure is predicted first and parameters are regressed afterward, suggesting the advantage depends on architectural context.

## Generation architectures from transformers to diffusion to LLMs

The generation machinery spans four architectural families, each with distinct strengths for different aspects of CAD.

**DeepCAD's deterministic autoencoder** is notably *not* a VAE—there is no KL divergence or reparameterization trick. The encoder (4 transformer layers, 8 heads, FF dimension 512, embedding dimension 256, dropout 0.1) processes the 60-token embedding sequence and produces a **256-dim latent vector z via average pooling** across all output positions. The decoder shares the same hyperparameters but operates **non-autoregressively**: it takes learned constant embeddings (similar to DETR) and attends to z via cross-attention, predicting all 60 commands in parallel. Two output heads produce command types (6-class softmax) and parameters (16 × 256-class classification). The loss is pure cross-entropy: L = Σ ℓ(t̂_i, t_i) + **λ=2** · Σ Σ ℓ(p̂_{i,j}, p_{i,j}), ignoring padding and unused parameters. Since the autoencoder's latent space lacks a tractable sampling distribution, a separate **WGAN-gp** (4-layer MLP generator mapping 64-dim noise → 256-dim latent; 4-layer MLP discriminator; critic iterations per generator step: 5; gradient penalty weight: 10) is trained for **200K iterations** on encoded latent vectors to enable unconditional generation. This two-stage design achieves 99.4% command accuracy and 97.5% parameter accuracy in reconstruction.

**BrepGen's structured diffusion** uses 4 sequential transformer-based denoisers in a top-down cascade, each conditioned on the outputs of preceding stages. The factored probability is p(F_p) · p(F_z|F_p) · p(E_p|F) · p(E_zv|F, E_p): first face bounding boxes (6D each), then face latent codes (48D each), then edge bounding boxes (6D each), then joint edge-vertex latents (18D each). Each denoiser has **12 self-attention layers with 12 heads**, using pre-LayerNorm transformers with sinusoidal timestep embedding. Training follows standard DDPM with **T=1000 timesteps** and linear noise schedule, optimizing the L2 noise prediction loss ‖ε − ε_θ(x_t, t)‖². At inference, face and edge geometry denoisers use **PNDM with 200 steps** for speed, while position denoisers use full DDPM sampling for precision. Face denoisers train for **3,000 epochs** (batch 512) and edge denoisers for **1,000 epochs** (batch 64), using AdamW with learning rate **5×10⁻⁴**. Topology is encoded implicitly through **mating duplication** (shared edges/vertices duplicated under each parent face) and **association duplication** (padding to fixed branching factors—50 faces per solid, 30 edges per face for DeepCAD shapes). The maximum is 80 faces and 40 edges per face for the Furniture/ABC datasets.

**SkexGen's parallel codebook architecture** trains three independent transformer encoders (topology, geometry, extrusion—all 4 layers, 8 heads) that each compress their respective subsequences into a small set of code-tokens, quantized against the corresponding codebook. The sketch decoder is an autoregressive transformer with causal masking that cross-attends to the concatenation of topology (4 codes) and geometry (2 codes) code-tokens, predicting among 4,101 classes (4,096 pixel locations + 4 end-primitive types + 1 end-sequence). The extrude decoder predicts among 72 classes. Sketch and extrude branches train independently for **500 epochs** (batch 128).

**Graph neural networks** provide the discriminative backbone for B-rep understanding. **UV-Net** (Jayaraman et al., CVPR 2021) couples 2D CNNs on face UV-grids with 1D CNNs on edge UV-grids, storing features as node and edge attributes in a face-adjacency graph processed by message-passing GNNs. **BRepNet** (Lambourne et al., CVPR 2021) defines convolutions on oriented coedges (halfedges), using topological walks—sequences of navigation operations (next, previous, mating coedge, parent face)—as convolutional kernels. With only ~360K parameters, BRepNet achieves 77.1% IoU on the Fusion 360 segmentation dataset. UV-Net's representation directly inspired BrepGen's 32×32 UV-grid sampling strategy.

## Reconstructing valid solids from neural predictions

The gap between raw neural output and a valid STEP file is where most systems fail, and the reconstruction pipeline differs dramatically between approaches.

**Command sequence execution** (DeepCAD, SkexGen, Text2CAD) parses the predicted command sequence and executes it in **OpenCASCADE**. Each ⟨SOL⟩ begins a loop; line/arc/circle commands construct 2D wire geometry; the extrude command creates a 3D solid with specified boolean operation. When commands are geometrically consistent, this produces valid B-rep solids directly. DeepCAD reports a **3.3% invalid ratio** on reconstructed autoencoded sequences, rising significantly for unconditionally generated samples.

**BrepGen's topology recovery** is far more complex. After denoising all four levels, decoded point grids are converted to parametric geometry using `GeomAPI_PointsToBSplineSurface` (faces) and `GeomAPI_PointsToBSpline` (edges) from OpenCASCADE. Then mating deduplication merges edges and vertices with similar features across different parent faces using thresholds of **~0.08 for bounding box distance** and **~0.2 for shape feature distance** (found via grid search). Association deduplication removes duplicate children within each parent. Merged edges form closed wire loops that trim the fitted B-spline surfaces. Finally, trimmed faces are **sewn together** using OpenCASCADE's sewing algorithms to produce watertight solids. BrepGen achieves **~62.9% validity** on DeepCAD shapes. The newer **HoLa** (SIGGRAPH 2025) eliminates edges and vertices from the latent space entirely, deriving them from surface primitives via a neural intersection network, and achieves **82.7% validity**—the current state-of-the-art for direct B-rep generation.

**Constraint solving** addresses the related problem of producing fully-constrained parametric sketches. The Autodesk Research team (Casey et al., 2025) demonstrated that **RLOO reinforcement learning alignment** of the Vitruvion constraint generation model achieves **93.05% fully-constrained sketches**, compared to 34% with naive supervised fine-tuning and 8.87% without any alignment. The constraint solver (Newton-Raphson iteration over geometric constraint equations) provides reward signals—solvability, stability, fully/over-constrained status—that the RL algorithm optimizes. GRPO achieved a close **91.59%**. This work trains on the SketchGraphs dataset (~2.8M sketches, only 8.27% naturally fully-constrained), showing that RL can dramatically improve constraint satisfaction even when training data is mostly under-constrained.

**The topology gap remains the hardest problem** because B-rep topology requires strict discrete combinatorial structure: each edge connects exactly two vertices, edges on each face form closed loops, every edge is shared by exactly two faces (manifold condition), and faces sew together without gaps. Even small errors—a single missing edge—break watertightness entirely. Neural networks naturally output continuous values, making this discrete constraint satisfaction fundamentally difficult. The problem is fragile (small errors are catastrophic), variable (topology varies enormously across shapes), and combinatorially explosive (the space of valid topologies is sparse within all possible adjacency configurations).

## The 2024–2025 wave changes everything

The period from late 2024 through 2025 brought a paradigm shift: pretrained LLMs replaced bespoke transformers, reinforcement learning became standard, and direct B-rep generation matured rapidly.

**STEP-LLM** (January 2025) pioneered direct generation of industry-standard STEP files by LLMs. The key insight is **DFS reserialization**: STEP files encode B-rep as a DAG of cross-referenced entities that, in raw form, scatter related entities far apart. DFS traversal of the parsed hierarchy linearizes the graph while preserving locality—related entities appear nearby in the sequence. The system uses **Llama-3.2-3B-Instruct** and **Qwen-2.5-3B** as backbones, trained on ~40K STEP-caption pairs (captions generated via GPT-4o from 9-view renders). Chain-of-thought structural annotations guide global coherence, RAG enriches training context, and RL with geometry-aware rewards refines fidelity. The combination achieves **>95% renderability**.

**FlexCAD** (ICLR 2025, Microsoft) fine-tunes **LLaMA-3-8B** with a hierarchy-aware masking strategy for controllable CAD generation across all construction hierarchies (sketch-extrusion, extrusion, sketch, face, loop, curve). CAD models are converted to structured text with different hierarchy fields masked at each epoch, enabling a single unified model to handle generation at any level. **CAD-Llama** (CVPR 2025) similarly adapts open-source LLMs using a Structured Parametric CAD Code (SPCC) format with adaptive pretraining and instruction tuning. **CADFusion** (ICML 2025) alternates between Sequential Learning on ground-truth parametric sequences and Visual Feedback stages where rendered outputs are scored by vision-language models, directly addressing the many-to-one mapping between parametric sequences and visual appearance.

On the B-rep front, **BrepDiff** (SIGGRAPH 2025) introduces **single-stage B-rep diffusion** with masked UV grid representations and asynchronous shifted noise schedules, eliminating BrepGen's complex four-stage pipeline while enabling intuitive editing operations. **DTGBrepGen** (CVPR 2025) explicitly decouples topology generation (via transformer encoder-decoder) from geometry generation (via 4 conditional diffusion models), attacking the topology gap by generating valid topological structures first. **BrepGPT** (SIGGRAPH Asia 2025) uses a Voronoi Half-Patch representation that decomposes B-reps into unified local units encoded via dual VQ-VAEs for single-stage autoregressive generation.

## Code generation sidesteps topology entirely

The code generation pathway—text → LLM → CadQuery/OpenSCAD/KCL → CAD kernel → STEP file—eliminates the topology gap by construction. The LLM only needs to produce syntactically correct procedural instructions; OpenCASCADE's ~2.5 million lines of C++ handle all topological consistency, shape healing, and boolean operations automatically.

**GPT-4o achieves over 90% first-attempt execution rates** on CadQuery generation. Fine-tuned models do even better: **Text-to-CadQuery** (Xie et al., 2025) trained six open-source LLMs on 170K CadQuery annotations generated by Gemini 2.0 Flash for just $170. Qwen2.5-3B achieved median Chamfer Distance of 0.191 with ~1.3% invalid rate for Mistral 7B. Even the tiny CodeGPT-small (124M parameters) outperformed the Text2CAD baseline, demonstrating that CadQuery as output format is inherently more effective than task-specific command sequences. **CAD-Coder** (Guan et al., 2025) achieved **100% valid syntax** by fine-tuning a VLM with two-stage SFT + GRPO reinforcement learning on 163K image-code pairs. The **CADCodeVerify** system (ICLR 2025) adds visual verification: after compilation, a VLM generates validation questions about the object from the original prompt, answers them using rendered reference images from 4 viewpoints, and feeds discrepancies back as corrective feedback.

**AIDL** (Jones et al., MIT CSAIL, Pacific Graphics 2025) takes a fundamentally different approach by designing a language *for* LLMs rather than adapting LLMs to existing languages. AIDL is a solver-aided hierarchical DSL where the LLM specifies declarative geometric constraints (coincident, equal, symmetric) rather than computing exact coordinates. A **SolveSpace**-based backend resolves constraints via recursive post-order tree traversal with iterative deepening. Without constraints, parts become detached because relative positions cannot be enforced. Without hierarchy, modifications require understanding the entire model. AIDL achieved higher CLIP scores than OpenSCAD despite the LLM never seeing AIDL in training data—validating the core thesis that designing languages around LLM capabilities yields better results than forcing LLMs to learn existing CAD APIs.

**Zoo.dev's KCL** (KittyCAD Language) is the commercial embodiment of code-first CAD. KCL is a functional programming language where the code *is* the model—no hidden binary state. Variables are immutable; geometry replication uses higher-level patterns rather than loops. This functional purity enables caching, instancing, and bidirectional linking between code and 3D viewer (clicking an edge highlights the source line). Zoo's proprietary geometry engine (not OpenCASCADE) produces B-Rep models exportable as STEP files. The Text-to-CAD pipeline feeds natural language through Zoo's ML model to generate KCL code executed by this engine.

Code generation excels at **prismatic, mechanical geometry**—extrusions, cuts, holes, chamfers, fillets—but struggles with freeform organic shapes where NURBS control point arrays are unintuitive to specify procedurally. Token length limits constrain complex models, and LLMs have difficulty maintaining spatial consistency across many components in large assemblies.

## Conclusion

The field has converged on a clear division of labor. **For mechanical parts with well-defined parametric structure**, LLM-based code generation (CadQuery, KCL, AIDL) delivers near-perfect validity by leveraging CAD kernel guarantees, with reinforcement learning from geometric rewards (CAD-Coder, CADFusion, cadrille) emerging as the key technique for pushing accuracy beyond what supervised fine-tuning alone achieves. **For freeform geometry and complex B-rep topology**, direct neural generation remains necessary but has improved dramatically: HoLa's holistic latent approach reaches 83% validity versus BrepGen's 63%, and single-stage methods like BrepDiff eliminate multi-stage pipeline complexity. The most consequential architectural insight across both pathways is that **quantization-as-classification outperforms regression** for geometric parameters when the network must independently predict each parameter, because classification naturally produces multi-modal distributions that respect discrete geometric constraints. The most consequential training insight is that **RL alignment with solver/kernel feedback** consistently unlocks 20–60 percentage point improvements in constraint satisfaction and validity—from 34% to 93% for sketch constraints, from supervised-only baselines to 100% compilation rates for code generation. The topology gap remains open for direct neural approaches, but the combination of topology-geometry decoupling (DTGBrepGen), neural intersection (HoLa), and autoregressive B-rep tokenization (BrepGPT, AutoBrep) is closing it rapidly.
